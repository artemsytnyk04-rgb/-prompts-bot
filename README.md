# Бот-библиотека промптов — инструкция по запуску

## Что это
Telegram-бот, который выдаёт готовые промпты по кнопкам (отдел → промпт) или по
поиску словом (например «возражение», «негатив», «выгорание»). Прямо в Telegram,
без необходимости открывать Notion.

Сейчас в базе **16 промптов** — полностью реальные, из разделов «Продажи» (6) и
«Поддержка» (10). Это рабочий прототип: логика проверена (поиск, разбивка длинных
сообщений, кнопки — см. `bot.py`), осталось добавить остальные 4 отдела (57 промптов)
по этому же шаблону в `data.py`.

## Шаг 1 — получить токен бота (5 минут, бесплатно)
1. Откройте Telegram, найдите **@BotFather**.
2. Отправьте `/newbot`, придумайте имя и username (должен заканчиваться на `bot`).
3. BotFather пришлёт токен вида `123456789:AAH...` — сохраните его, это ключ от бота.

## Шаг 2 — где запустить (варианты по возрастанию сложности)

### Вариант А — Railway.app (рекомендую, бесплатный лимит хватает с запасом)
1. Зарегистрируйтесь на railway.app через GitHub.
2. Загрузите папку `tgbot/` (bot.py, data.py, requirements.txt) в новый репозиторий GitHub.
3. В Railway: New Project → Deploy from GitHub repo → выберите репозиторий.
4. В настройках проекта (Variables) добавьте переменную `BOT_TOKEN` = ваш токен из Шага 1.
5. Railway сам определит Python-проект и запустит `python bot.py`. Готово — бот работает 24/7.

### Вариант Б — свой сервер/VPS
```bash
pip install -r requirements.txt
export BOT_TOKEN="ваш_токен_сюда"
python bot.py
```
Чтобы бот не падал при перезагрузке сервера, запустите его через `systemd`,
`pm2` или в `screen`/`tmux` — если нужно, могу расписать конкретный вариант.

### Вариант В — быстро протестировать у себя на компьютере
```bash
pip install -r requirements.txt
export BOT_TOKEN="ваш_токен_сюда"   # на Windows: set BOT_TOKEN=ваш_токен
python bot.py
```
Пока скрипт запущен — бот отвечает. Закроете терминал — бот перестанет отвечать
(для теста этого достаточно, для боевого использования нужен Вариант А или Б).

## Шаг 3 — проверить
Напишите вашему боту в Telegram `/start` — должно появиться меню отделов.

## Как добавить остальные 4 отдела (57 промптов)
Откройте `data.py` — там уже есть `sales` и `support`. Добавьте новые ключи
`marketing`, `ceo`, `product`, `automation` по такому же шаблону:
```python
"marketing": {
    "label": "📣 Маркетинг",
    "prompts": [
        {
            "id": "m1",
            "title": "...",
            "pain": "...",
            "task": "...",
            "prompt": "...",
            "howto": "...",
        },
        # ещё промпты
    ],
},
```
Данные для всех 73 промптов у меня уже есть (я их разбирал для Notion-базы) —
если хотите, в следующем сообщении просто напишите «добавь остальные отделы»,
и я пришлю готовый дополненный `data.py` целиком.

## Ограничения этой версии (сознательно, чтобы не переусложнять MVP)
- Данные промптов хранятся прямо в `data.py`, а не подтягиваются из Notion
  автоматически. Если поменяете промпт в Notion — нужно вручную поправить и в
  `data.py`. Для 73 более-менее стабильных промптов это нормально; если начнёте
  редактировать базу часто — есть смысл подключить Notion API напрямую (это уже
  следующий уровень сложности, могу спроектировать отдельно).
- Бот не запоминает, кто его открывал, и не считает статистику использования —
  это тоже можно добавить (простая таблица в SQLite), но для первой версии не
  обязательно.
